2주동안

-역할 : 동적으로 변화시키는 것 (이벤트를 만들어서 등록하는 언어) (이벤트 : 유저의 특정행동에 따라서, 동작하는 코드)
 ㄴjs부터는 수학적인 로직이 들어감(에러나 버그가 나타남)
 ㄴ매우매우 각광 받는 언어
 ㄴ숙련도가 100퍼 이상 되어야 함

-디자인 패턴 : 코드를 작성하는 스타일
  ㄴjs는 크게 2가지의 디자인 패턴이 존재
 1. 객체형 프로그래밍 (전통 강호) class
 2. 함수형 프로그래밍 (신흥 강자)

-함수
  ㄴ고등학교 때 배운 함수의 정의 (x값을 넣으면 y값이 나오는 수식)
 ㄴ프로그래밍에서의 함수 (매개변수를 넣으면 return값이 나오는 코드 다발)

-함수의 장점
 1. 중복 코드를 제거 (중복코드는 안 좋음)
  2. 가독성이 증가 (volume of speer)
 3. 에러와 버그 찾기에 용이해짐
  *함수의 역할은, 코드를 논리적으로 분할시키는 역할 (구역을 나누는 것)
